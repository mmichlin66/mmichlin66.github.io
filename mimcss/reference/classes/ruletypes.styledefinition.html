Class StyleDefinition<P, O>

The StyleDefinition class is a base for all classes that contain defininitions of CSS rules.

Type parameters

  • P: StyleDefinition = any

    Parent style definition class. Parent of top-lvel class is null.

  • O: StyleDefinition = any

    Top-level style definition class, which is the owner of this class. The top-level class is its own owner.

constructor

  • new StyleDefinition<P, O>(parent?: P): StyleDefinition<P, O>

  • Style definition instances are created directly only by the styled components - that is, components that use different styles for each instance. Otherwise, style definition instances are created when either the $use, $embed or activate function is called.

    Type parameters

    • P: StyleDefinition<any, any, P> = any

    • O: StyleDefinition<any, any, O> = any

    Parameters

    • Optional parent: P

      Reference to the parent style definition class

Accessors

$owner

  • get $owner(): undefined | O

  • Refers to the instance of the style definition class which is the owner of this style definition object. The owner is the top-level class in the chain of style definition classes. Through this member, all rules and other members defined in the owner definition class can be accessed. For top-level style definitions, this property points to itself. This property can be undefined if the parent instance was not provided when creating the style definition class manually.

    Returns undefined | O

$parent

  • get $parent(): undefined | P

  • Refers to the instance of the style definition class which is the parnt of this style definition object in the chain of style definition classes. Through this member, all rules and other members defined in the parent definition class can be accessed.For top-level style definitions, this property is always undefined. This property can also be undefined if it was not provided to the constructor when creating the style definition class manually.

    Returns undefined | P

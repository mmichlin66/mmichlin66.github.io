Interface ISelectorBuilder

Provides means to build complex selectors from multiple selector items of all possible kinds including tags, classess, IDs, attributes, pseudo classes and pseudo elements combined with CSS combinators.

has

  • has(...items: CssSelector[]): ISelectorBuilder
  • has(combinator: "" | SelectorCombinator, ...items: CssSelector[]): ISelectorBuilder

  • Adds the ":has" pseudo class to immediately follow the existing selector. If multiple items are specified, they are interpreted as a list; that is, they are combined using the "," combinator.

    Example:

    class MyStyles extends css.StyleDefinition
{
    c1 = css.$class({...})
    c2 = css.$class({...})

    // produces css: section:has(.c1, .c2) {...}
    s1 = css.$style( css.sel("section").has( this.c1, this.c2), {...})
}

    Parameters

    • Rest ...items: CssSelector[]

      List of selector items to be added

    Returns ISelectorBuilder

  • Adds the ":has" pseudo class with partial selector that starts with the given combinator. If multiple items are specified, they are concatenated using the given combinator. A special value "" (empty string) of the combinator parameter allows to concatenate the items without any combinator.

    Example:

    class MyStyles extends css.StyleDefinition
{
    c1 = css.$class({...})
    c2 = css.$class({...})

    // produces css: section:has(> .c1 > .c2) {...}
    s1 = css.$style( css.sel("section").has( ">", this.c1, this.c2), {...})

    // produces css: section:has(.c1.c2) {...}
    s1 = css.$style( css.sel("section").has( "", this.c1, this.c2), {...})
}

    Parameters

    • combinator: "" | SelectorCombinator

      Combinator to use to concatenate the items and to attach them to the existing selector.

    • Rest ...items: CssSelector[]

      List of selector items to be added

    Returns ISelectorBuilder

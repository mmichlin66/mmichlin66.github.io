activate

  • activate<T>(instanceOrClass: T | IStyleDefinitionClass<T>, schedulerType?: undefined | number): T | null
    • Defined in api/SchedulingAPI.ts:17

    Activates the given style definition class or instance and inserts all its rules into DOM. If the input object is not an instance but a class, which is not yet associated with an instance, the instance is first created and processed. Note that each style definition instance maintains a reference counter of how many times it was activated and deactivated. The rules are inserted into DOM only upon first activation.

    Type parameters

    • T: StyleDefinition

    Parameters

    • instanceOrClass: T | IStyleDefinitionClass<T>
    • Optional schedulerType: undefined | number

    Returns T | null

deactivate

  • deactivate(instance: StyleDefinition, schedulerType?: undefined | number): void
    • Defined in api/SchedulingAPI.ts:35

    Deactivates the given style definition instance by removing its rules from DOM. Note that each style definition instance maintains a reference counter of how many times it was activated and deactivated. The rules are removed from DOM only when this reference counter goes down to 0.
